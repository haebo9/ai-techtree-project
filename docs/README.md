# 📂 Documentation Index

프로젝트의 기획(PRD), 설계(Design), 참고 자료(Knowledge)를 관리하는 공간입니다.

## 📘 1. prd (기획)
**"무엇을, 왜 만드는가?"** (요구사항 및 서비스 흐름)

* `product_spec.md`: 핵심 기능 명세 (GitHub 코드 분석 및 점수 산출)
* `personas.md`: 타겟 유저 정의 (이직 희망 개발자, 채용 담당자)
* `user_flow.md`: 서비스 흐름도 (URL 입력 -> 분석 -> 결과 리포트)

## 📐 2. design (설계)
**"어떻게 구현하는가?"** (아키텍처 및 데이터 구조)

* `architecture.md`: 시스템 전체 구조 (Next.js - FastAPI - LangGraph)
* `agent_workflow.md`: AI 에이전트 동작 흐름 (Fetcher -> Analyzer -> Scorer)
* `db_schema.md`: MongoDB 데이터 모델 (User, Report 컬렉션 구조)

## 📚 3. knowledge (지식)
**"무엇을 참고하고 배웠는가?"** (기술 의사결정 및 트러블슈팅)

* `tech_decisions.md`: 기술 스택 선정 이유 (Why LangGraph? Why Atlas?)
* `troubleshooting.md`: 개발 중 마주친 에러와 해결 과정 기록
* `references.md`: 주요 참고 문서 및 튜토리얼 링크 모음
