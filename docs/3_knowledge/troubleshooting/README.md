# Troubleshooting Log

> 개발 과정에서 발생한 이슈, 에러, 그리고 해결 과정을 기록하는 공간입니다.
> 동일한 실수를 반복하지 않고, 팀원들과 지식을 공유하기 위해 작성합니다.

## 📝 작성 가이드
1. 새로운 이슈가 발생하면 `ts_template.md`를 복사하여 `YYYY-MM-DD_issue_name.md` 파일을 생성합니다.
   - [템플릿 바로가기](./ts_template.md)
2. 문제의 현상, 원인, 해결 방법을 상세히 기록합니다.
3. 아래 **Issue List**에 해당 문서를 링크합니다.

## 🏷️ Status Guide
| Icon | Status | Description |
| :--- | :--- | :--- |
| ✅ | **Solved** | 원인 규명 및 해결 완료 |
| 🚧 | **In Progress** | 현재 원인 분석 또는 해결 시도 중 |
| ⏭️ | **Deferred** | 당장 해결하지 않고 보류 (우선순위 낮음) |
| ❌ | **Won't Fix** | 기술적 제약 또는 기획 의도에 의해 수정하지 않음 |
